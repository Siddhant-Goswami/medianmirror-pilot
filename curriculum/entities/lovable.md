## Overview
Lovable is an AI-powered UI prototyping and MVP generation tool that takes a Product Requirements Document (PRD) and generates a working frontend application. In the 100x Cohort 7 curriculum, Lovable is the first tool in the 9-step ship cycle — it takes the PRD and outputs a deployable frontend MVP.

## Key Contributions / Role
- Generates full frontend applications from text/PRD input
- Produces code that can be exported to GitHub for further development
- Occupies the "ideate to prototype" phase in the ship cycle
- Compared to Cursor: Lovable = greenfield generation; Cursor = codebase refinement
- Part of the "vibe coding" toolkit alongside Bolt, Rork, and Emergent

## Notes
The GIGO principle (Garbage In, Garbage Out) applies directly: Lovable output quality is entirely determined by PRD quality. A vague PRD produces a generic MVP. A specific PRD with user flows, data entities, and component descriptions produces a usable prototype.
